API Endpoints

Complete REST API Specification

Endpoint Method Description Authentication
/api/auth/register POST Create new user account with validation ❌ Public
/api/auth/login POST Authenticate user and return JWT token ❌ Public
/api/users/me GET Retrieve authenticated user's profile ✅ JWT Required
/api/users/me PUT Update authenticated user's information ✅ JWT Required
/api/users/me DELETE Permanently delete user account ✅ JWT Required
Base URL: http://localhost:5000
Content-Type: application/json
Authentication: Bearer {jwt_token}

Security Implementation

Enterprise-Grade Security Measures

🔒
Password Security
Bcrypt hashing with 10 salt rounds ensures passwords are never stored in plain text. Computational complexity prevents brute-force attacks.
🎫
JWT Authentication
Stateless tokens with 24-hour expiration. Signed with secret keys to prevent tampering. Bearer token authentication for protected endpoints.
🛡️
Security Middleware
Helmet.js adds HTTP security headers. Rate limiting prevents abuse. Input validation and sanitization protect against injection attacks.
🚫
Anti-Enumeration
Generic error messages prevent user enumeration attacks. "Invalid credentials" for both wrong email and wrong password.

System Architecture

How All Components Work Together

👥 Client Applications
🌐 Swagger UI - Interactive API Explorer
📮 Postman - API Testing Tool
⚛️ Frontend Apps - Web/Mobile Clients
🔌 External Services - Third-party Integration
↓ HTTP Requests (JSON) ↓
⚙️ Express.js Web Server
🛣️ Routes Layer
• /api/auth/register → authRoutes.js
• /api/auth/login → authRoutes.js
• /api/users/* → userRoutes.js
🎮 Controllers Layer
• authController.js → Login & Registration Logic
• userController.js → Profile Management & CRUD Operations
🔒 Middleware Layer
• auth.js → JWT Token Verification & Validation
• validation.js → Input Sanitization & Schema Validation
• helmet.js → Security Headers & CORS Configuration
📊 Models Layer
• user.js → Database Operations & ORM Integration
↓ SQL Queries ↓
💾 SQLite Database
📋 users Table Schema
• id (INTEGER, PRIMARY KEY)
• username (TEXT, UNIQUE)
• email (TEXT, UNIQUE)
• full_name (TEXT)
• password_hash (TEXT, bcrypt encrypted)
• created_at (TIMESTAMP)
• updated_at (TIMESTAMP)
Architecture Benefits:
• Separation of Concerns: Clear layer responsibilities
• Security: Multiple validation and protection layers
• Testability: Each layer can be tested independently
• Scalability: Easy to add new features and endpoints
